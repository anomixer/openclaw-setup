> **Founder's Confession**: In a recent Feb 26 interview, Peter Steinberger bluntly stated: "OpenClaw has the most thorough privacy solution available right now (data never leaves local)... **but security is another story.**"

### 🛡️ Incidents & Vulnerabilities
- **Massive CVE Outbreaks (2026-02-26)**:
  - 💥 **CVE-2026-25253 (Critical RCE)**: CVSS 8.8. A WebSocket token leak allowing 1-click full gateway control. Affects <= v2026.1.29.
  - 💉 **CVE-2026-26323 (Command Injection)**: Attackers used malicious Git commit author emails to achieve RCE against devs/CI running the `update-clawtributors.ts` script.
  - 🔓 **CVE-2026-26326 (Information Leak/CWE-200)**: Low-privilege users can abuse the `skills.status` API to extract Discord Bot Tokens.
- **Meta Internal Dumpster Fire (2026-02-25)**: A Meta Director's OpenClaw bot wiped 200+ emails from her inbox. Experts mocked OpenClaw as a "Security Dumpster Fire." Meta instituted a company-wide ban.
- **Npm Supply Chain Attack (2026-02-17)**: Hackers compromised the `Cline` npm tool, secretly installing OpenClaw as a backdoor Trojan. Over 4,000 developers were affected.

### 🕵️ Infostealers & Exposures
- **AMOS macOS Stealer (Recent)**: Trend Micro revealed malicious Skills actively distributing new Atomic macOS Stealer (AMOS) variants. Hudson Rock tracked the first credential theft via `config` files (OC-09).
- **Vidar Infostealer Threat (2026-02-16)**: **Vidar** malware began specifically targeting OpenClaw (`config.toml`, `tokens.json`). Astrix Security released Enterprise scanners to hunt down rogue OpenClaw footprints.
- **135K Exposed Instances (2026-02-12)**: Security scans discovered 135,000 OpenClaw instances exposed completely unprotected on the public internet.

### 🚨 Malicious Skills & Corporate Bans
- **Snyk Report**: 7.1% of skills leak sensitive data; Crypto skills are the most deadly (23% malicious).
- **Official Defense**: Implementation of a Skill Safety Scanner and reputation framework dropped malicious installs by 73%.
- **Corporate Blacklists**: Cisco officially called it a "Security Nightmare". Western banks and Meta fully banned employees from installing it on work machines—resulting in a massive underground "Shadow AI" culture.
