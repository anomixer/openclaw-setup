## 🏢 企業的擔憂

### Cisco 官方警告（2026-02-10）

**標題**: "Personal AI agents like OpenClaw are a security nightmare"

**核心論點**：
1. 🔓 **權限過大** — AI agent 可存取檔案、執行指令、讀取記憶體
2. 🕵️ **資料外洩** — 對話記錄可能包含企業機密

4. 🌐 **供應鏈攻擊** — Skills registry 成為新的攻擊面

**Cisco 建議**：企業 ❌ 禁用 OpenClaw，或 ⚠️ 部署在隔離網路、禁用 skills、記錄所有對話、定期安全稽核

### ✅ 已修復問題

| 問題 | 狀態 | 解法 |
|------|------|------|
| `spawn EINVAL`（安裝 Plugin 失敗）| ✅ v2026.2.5+ 已修復 | 升級到最新版即可 |
| `unknown integration: openclaw` | ✅ 已修復 | 升級 Ollama 至 v0.15.4+ |

### ⚠️ 持續追蹤

**本地模型 Memory 功能不穩定**：
- 現象：Ollama 本地模型記憶功能時好時壞（#8470 / #8131）
- 對策：升級 Ollama ≥ v0.15.5、升級 OpenClaw ≥ v2026.2.8、安裝 `supermemory` plugin、等待 2 月底大更新

### Windows 平台穩定度

| 功能 | Windows 原生 | WSL2 | 建議 |
|------|-------------|------|------|
| 基本對話 | ✅ 穩定 | ✅ 穩定 | 都可以 |
| Memory | ⚠️ 不穩定 | ✅ 穩定 | **用 WSL2** |
| Skills | ⚠️ 部分相容 | ✅ 完全相容 | **用 WSL2** |
| Gateway | ✅ 穩定 | ✅ 穩定 | 都可以 |
| 安全掃描 | ✅ 正常 | ✅ 正常 | 都可以 |

> **結論**：如果你需要完整功能，**強烈建議使用 WSL2**。
